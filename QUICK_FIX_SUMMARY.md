## 🔧 작동 확인 방법

### 1. 서버 재시작
```bash
# 기존 서버 중지 (Ctrl+C)
npm run dev -- -p 5000
```

### 2. 브라우저 캐시 클리어
- Chrome: Ctrl+Shift+Delete
- 또는 시크릿 모드로 테스트

### 3. 테스트 시나리오

#### A. 로그인 테스트
1. http://localhost:5000 접속
2. "시작하기" 또는 "로그인" 클릭
3. 회원가입: 이메일, 비밀번호, 이름 입력
4. ✅ 자동으로 `/dashboard`로 이동
5. ✅ 페이지 새로고침해도 로그인 유지
6. ✅ 텍스트가 선명하게 보임

#### B. 플래시카드 테스트
1. 좌측 메뉴 "플래시카드" 클릭
2. "+ 새 플래시카드" 버튼 클릭
3. 앞면/뒷면 입력 후 저장
4. ✅ 카드 목록에 표시됨
5. "복습하기" 메뉴로 이동
6. ✅ 생성한 카드가 복습 대상에 표시

#### C. 커리큘럼 생성 테스트
1. "커리큘럼" 메뉴 클릭
2. "+ 새 커리큘럼" 클릭
3. 주제: "자바스크립트"
4. 학습 목표: "자바스크립트 기초를 배우고 싶습니다"
5. AI 모델 선택 (API 키 설정 필요!)
6. "커리큘럼 생성하기" 클릭
7. ✅ 10-30초 후 커리큘럼 생성됨
8. "학습 시작" 클릭하여 상세 페이지 확인

#### D. AI 개념 생성 테스트
1. 커리큘럼 상세 페이지에서
2. 각 토픽의 "개념 설명" 버튼 클릭
3. ✅ AI가 상세한 설명 생성

## ⚠️ 주의사항

### AI API 키 필수!
커리큘럼 생성 및 개념 설명 기능을 사용하려면 `.env.local`에 최소 하나의 AI API 키가 필요합니다:

```env
# 최소 하나는 설정하세요!
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_AI_API_KEY=...
```

### MongoDB 필수!
```bash
# MongoDB가 실행 중이어야 합니다
mongod
# 또는 MongoDB Atlas 사용
```

## 🐛 여전히 문제가 있다면?

1. **로그인이 안 됨**
   - 브라우저 쿠키 확인 (F12 > Application > Cookies)
   - `memora_auth` 쿠키 존재 여부 확인
   - 로그아웃 후 재로그인으로 세션 재발급

2. **페이지가 비어있음**
   - 콘솔 에러 확인 (F12 > Console)
   - MongoDB 연결 확인
   - 서버 재시작

3. **커리큘럼 생성 실패**
   - AI API 키 확인
   - API 사용량 한도 확인
   - 네트워크 연결 확인

4. **텍스트가 여전히 안 보임**
   - 브라우저 캐시 완전 삭제
   - 시크릿 모드로 테스트
   - CSS 적용 확인 (F12 > Elements)
